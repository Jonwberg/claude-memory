## This Hypothesis

**Question:** What facts from today will I reliably have access to in the next session?

**Why this matters:** Defines the boundary of what the memory system can actually preserve -- helps calibrate what to save.

**Prediction:** Only facts written to memory files before the Stop hook fires. In-context reasoning is lost.

## Test Plan

1. Write a specific test fact to semantic memory
2. End session (trigger Stop hook)
3. Start new session, check if fact is retrieved
4. Compare retrieved content to original

## Raw Results

- **Multi-session test completed (2026-04-05 -> 2026-04-06):** Test fact "The blue cardinal number is 7442" was written to semantic/test_fact_structured.md and indexed in MEMORY.md.
- **New session (2026-04-06):** Fact appeared in context via the claudeMd system-reminder at session start -- no explicit retrieval needed.
- **Mechanism:** MEMORY.md is injected as a context block at session start. Individual file CONTENTS are not auto-loaded -- only their one-line MEMORY.md descriptions are.
- **Bare fact file (not indexed):** Invisible in the new session -- confirms indexing is the gate, not file existence.
- **Refinement to prediction:** "Before Stop hook fires" is not the binding constraint. The binding constraint is "before MEMORY.md is updated with a pointer to the file." The Stop hook is the typical trigger for MEMORY.md updates, but wip-checkpoint.mjs now writes mid-session entries directly.

## Synthesis

CONFIRMED with important nuance. Facts written to files AND indexed in MEMORY.md reliably survive to the next session. In-context reasoning that was never written down is indeed lost. However, the prediction understated two things: (1) only the MEMORY.md one-liner description is loaded, not the full file content, so description quality is critical; (2) the Stop hook is not the only path -- any tool that writes to MEMORY.md mid-session (like wip-checkpoint.mjs) also preserves facts. The practical implication: write a strong one-liner description in MEMORY.md for any fact you want to survive. The file format matters less than the description quality.

**Outcome:** CONFIRMED
