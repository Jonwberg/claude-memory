## This Hypothesis

**Question:** Has the ExitPlanMode cwd bug (GitHub #22343) been fixed in recent Claude Code releases, or is it still active?

**Why this matters:** If the bug is fixed, relative paths in ExitPlanMode hooks become safe. If still present, absolute paths are mandatory and the existing workaround must be maintained.

**Prediction:** The bug may or may not be fixed — version tracking is needed to determine status.

## Test Plan

1. Check the installed Claude Code version
2. Inspect settings.json for any existing ExitPlanMode hooks
3. Add a temporary ExitPlanMode hook that writes process.cwd() to a file, trigger it, observe output
4. Document findings and remove temporary hook

## Raw Results

**Environment:**
- Node.js version: v24.12.0
- Claude Code version: 2.1.92 (Claude Code)
- Claude executable: `C:\Users\Jon Berg\.local\bin\claude.exe`

**settings.json hook audit:**
Current hooks in `C:/Users/Jon Berg/.claude/settings.json`:
- `UserPromptSubmit`: retrieval-hook.mjs
- `PostToolUse` (Bash): error-capture-hook.mjs
- `PostToolUse` (Write|Edit): wip-checkpoint.mjs
- `Stop`: consolidation.mjs
- **No ExitPlanMode hook is currently configured.**

**Attempted live test:**
Adding a temporary ExitPlanMode hook and triggering it from an automated subagent is not feasible — ExitPlanMode requires a user to exit plan mode interactively, which cannot be triggered programmatically from within this session.

**Version-to-fix mapping:**
GitHub issue #22343 was not resolvable via the available tools in this test. The issue fix status for version 2.1.92 is unknown without access to the Claude Code changelog or GitHub.

**Inference from existing code:**
The memory system hooks (retrieval-hook.mjs, consolidation.mjs, etc.) all use `os.homedir()` to construct absolute paths rather than relying on `process.cwd()`. This design pre-emptively avoids the ExitPlanMode cwd bug regardless of fix status.

## Synthesis

The ExitPlanMode cwd bug cannot be confirmed as fixed or unfixed for version 2.1.92 without either: (a) access to the Claude Code changelog/GitHub release notes, or (b) an interactive test where a user exits plan mode and the hook output is observed. The live hook test was not possible to execute automatically.

However, the key operational question — "does the existing memory system need to be updated?" — has a clear answer: **no change needed**. All existing hooks use absolute paths via `os.homedir()`, which sidesteps the cwd bug entirely. The absence of any ExitPlanMode hook in settings.json means there is no current exposure.

The conservative recommendation stands: any future ExitPlanMode hook should use `$CLAUDE_PROJECT_DIR` or `os.homedir()`-constructed absolute paths, not relative paths, until the fix is confirmed and verified in production.

**Outcome:** INCONCLUSIVE (live trigger not possible; operational risk is mitigated by existing absolute-path design)
