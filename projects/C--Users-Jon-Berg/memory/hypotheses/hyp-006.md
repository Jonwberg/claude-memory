## This Hypothesis

**Question:** Can I infer a user's actual skill level from their prompt style reliably?

**Why this matters:** Calibrating explanation depth to skill level makes responses much more useful. Miscalibration wastes time.

**Prediction:** Prompt length, vocabulary specificity, and error description quality are signals — but all have false positives.

## Test Plan

1. Review last 10 sessions from memory
2. For each: note prompt style markers and skill level inferred
3. Check if inferred level matched actual knowledge shown later in session
4. Identify which markers were most/least reliable

## Raw Results
- **No automated tests were run**, and the test plan requires reviewing past sessions from memory — a capability Claude does not have (no persistent cross-session memory).
- However, the core question — whether prompt style reliably signals skill level — can be partially addressed through reasoning about known failure modes and architectural constraints.
- Claude processes each conversation independently, meaning any "skill inference" is rebuilt from scratch every turn, with no longitudinal calibration data.
- Known confounds are readily identifiable from first principles: experts sometimes write terse/sloppy prompts; novices sometimes paste polished prompts from tutorials or LLMs; non-native speakers may appear less expert due to language rather than domain knowledge; copy-pasted error messages reveal tooling, not understanding.

## Synthesis
The test plan as written is **unexecutable** because Claude lacks persistent memory across sessions and cannot review historical interactions. However, the core prediction — that vocabulary, prompt length, and error description quality are real but unreliable signals — is well-supported by reasoning about known failure modes. Terse prompts from experts look like under-specified prompts from novices; polished prompts from novices (often LLM-assisted) mimic expert communication. Without the ability to track calibration accuracy over time, Claude has no mechanism to learn which signals are most reliable in practice. The most honest conclusion is that within-session recalibration (updating inferences as the conversation progresses) is the only trustworthy strategy, and initial inferences should be held lightly.

**Outcome:** INCONCLUSIVE
